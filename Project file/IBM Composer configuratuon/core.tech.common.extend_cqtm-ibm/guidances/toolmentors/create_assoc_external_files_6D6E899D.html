Using file locations

You must create a file location record for each externally referenced file. A file location is associated with one asset registry record and cannot be shared across asset registries. To create a file location record or to create an association to an external file, you must use the Rational ClearQuest Client or the Rational ClearQuest Client for Eclipse.

A file location can be in an Eclipse project or a file system. The path for a file location must either be a shared location (UNC or SMB) or a location stored in a Rational ClearCase VOB.

An externally referenced file includes all of the following:
  • One or more test motivator files associated with a test plan or test case record
  • A single test script file associated with a test case or configured test case record
  • A test log file referenced by a test log record

For more information about file locations, see the Help system for the Rational ClearQuest Client or the Rational ClearQuest Client for Eclipse.
