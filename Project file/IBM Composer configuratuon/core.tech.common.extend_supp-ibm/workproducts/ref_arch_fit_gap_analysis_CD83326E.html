Артефакт: Reference Architecture Fit Gap Analysis
This artifact is an analysis of the similarities and differences between a reference architecture and the requirements of the new system.
Домены: Architecture
Назначение
This artifact is used as the basis for deciding whether a reference architecture should be used or not and indicates any gaps where the reference architecture does not support the system requirements.
Описание
Краткая схема

This work product typically contains:

  • Executive Summary - A brief summary of the analysis, the recommendation and major issues identified.
  • Introduction - Sets the context of this analysis and identifies the reference architecture that is being used to compare against.
  • Key Drivers - Summarize the business drivers for the system.
  • Architecture Requirements Checklist - The major requirements (functional and non-functional) to be considered when performing the fit-gap analysis.
  • Subject Area Checklist: Subject areas, or domains, are the major subsystems present in the reference architecture. Content management, e-mail, enterprise application integration etc are examples of subject areas which may be present in a reference architecture.  One way of testing the "fit" to a reference architecture is to compare subject areas between the reference architecture and the new system, if they are known.
  • Statement of Fit: A list or table which describes the gaps between the reference architecture and the new system by comparing requirements and/or subject areas. Record here:
    • The requirement and/or subject area.
    • The degree of fit and how the system could be adapted if it is to use the reference architecture.
    • Costs and risks associated with adapting the system to use the reference architecture or not.

Доводка
Последствия отсутствияIf a reference architecture fit-gap analysis is not performed then an incorrect or inappropriate reference architecture may be selected or the costs associated with adapting a reference architecture may be underestimated.
Причины для отказа

It may not be appropriate to produce this artifact if:

  • The solution is in an area where no reference architectures exist or can be found.
  • The system being built is trivial and the use of a reference architecture would be too costly or otherwise inappropriate.
  • There is a clear statement made by the stakeholder that reference architectures should not be used.
  • The decision to use the reference architecture is straightforward and doesn't require a fit-gap analysis.
  • The fit will be evaluated through another approach, such as prototyping.
