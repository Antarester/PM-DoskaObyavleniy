Справочная таблица: Reference Architecture Fit-Gap Analysis
This checklist provides a set of questions that can be used as a guide when validating a Reference Architecture Fit-Gap Analysis.
Взаимосвязи
Связанные элементы
  • Reference Architecture Fit Gap Analysis
Элементы справочной таблицы
The criteria used to evaluate reference architectures has been approved by the stakeholders.
All major requirement areas have been considered.
The reference architecture can meet the critical dependencies in the target environment.

This includes the following:

  • The deployment of the architecture at all locations.
  • The integration of other existing (and planned) systems into the chosen architecture.
  •  Performance, security and scalability requirements.
Tradeoffs have been validated between the stakeholders and asset providers.
  • They meet business objectives for the system.
  • They sufficiently mitigate against risk.
  • They do not compromise the use of assets.  For example the assets are available within the published time scales, there are suitable service agreements in place etc.
The architecture is resilient to anticipated changes.
