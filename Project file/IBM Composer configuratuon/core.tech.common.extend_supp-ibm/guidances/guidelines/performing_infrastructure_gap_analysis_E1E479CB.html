Рекомендация: Performing Infrastructure Gap Analysis
This guideline describes a technique for performing an Infrastructure Gap Analysis.
Взаимосвязи
Связанные элементы
  • Infrastructure Gap Analysis
Основное описание

The Infrastructure Gap Analysis identifies the variance (gaps) between the IT organization's existing technology and application portfolios, the IT processes, and the IT skill sets and organization to those required to implement and support the new environment. The analysis includes the conclusions derived after a data gathering process. These conclusions quantify the impact of the transition required to migrate from the existing, "current", IT infrastructure to the desired, "future", IT infrastructure.  The Infrastructure Gap Analysis Artifact documents the work done to date, what was found during data gathering (high level requirements and key assumptions), the difference between the current and future infrastructure and the next steps needed to deal with the gaps.

Steps

Perform Gap Analysis

Develop the differences between the current infrastructure and the future infrastructure that must be addressed through planning.

  • Sort facts, assumptions, questions and issues (FAQI) by Technology, Process, People (TPP) in the FAQI worksheet (see Table 1).Group the Technical FAQI by server and network and assign an identifier to each FAQI. 
  • Group the Process FAQI by process (Asset Management, Backup Recovery, etc.); and assign an identifier to each FAQI.  Group the People FAQI by organization, staffing, training, and skills; and assign an identifier to each FAQI.
  • Sort all FAQI by the assigned group identifier within TPP.
  • Analyze FAQI data, along with data from the EUFG worksheet (see Table 2), to form findings by combining facts until they produce new thought, findings, that are pertinent to the Current or Future infrastructure.
  • Result:  First level analysis of facts (see Table 3).

Create conclusions/gaps from findings

  • Using the same process, analyze findings to form conclusions/gaps.
  • Review the conclusions/gaps using the entire client team to gain consensus.
  • Result:  Second level analysis of facts. 
  • Output: Infrastructure Gap Analysis with conclusions/gaps about the difference between the current infrastructure and the future infrastructure.

If the client does not have sufficient documentation to describe the existing infrastructure, the conclusions/gaps cannot be created.  Interviews with people knowledgeable about the current infrastructure will be required, along with consultants knowledgeable about infrastructure requirements at similar clients.  Combining this information can provide a generic list of gaps to be planned for.

Produce Gap Analysis Report

Create a report for review with the planning engagement executive sponsor to gain approval to proceed. An example report might contain:

  • Management Summary
  • Work Done to Date
  • High Level Requirements
  • Key Assumptions
  • Gaps Identified
  • Approach to Follow-On Implementation Planning
  • Appendix:  Detailed Findings
  • Result:  Consultant's view of the client situation, gaps and next steps - Example 4

Client review/revision of "strawman" report:

  • Put the entire report on the wall for client review, revision.
  • Adjust report to meet the client view.
  • Gain client consensus to the details of the report.
  • Result:  Client team consensus on content of the report.


Present to sponsoring executive:

  • Presentation
  • Apply revisions
  • Result:  Executive approval
  • Output: Final Infrastructure Gap Analysis
